Key Points from today

  • <span> and other inline elements stay on one line and cannot contain block elements. ids are referenced/accessed with '#'
  • <div> and other block level elements take up the whole line they are on. classes are referenced/accessed with '.'
  • HTML elements can only have 1 id but multiple classes
  • Remember only the most specific rules take precedent over others and then the last ones
  • As Orriel showed using percentages on her table, 100% means that it will take up that percentage of the space of whatever it is inside of. It is a bit more responsive than px or pixels are.
